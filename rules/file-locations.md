# JsonUI File Placement Rules

## JSON Layout Files

Place in `layouts_directory` from config (default: `Layouts`).

Subdirectories allowed:
```
{layouts_directory}/
├── home.json                    # Screen layout
├── settings.json                # Screen layout
├── home/                        # Screen-specific subdirectory
│   ├── item_cell.json           # Cell used only in home screen
│   └── banner_cell.json         # Cell used only in home screen
├── common/                      # Shared components across screens
│   ├── header.json              # Reusable header
│   └── footer.json              # Reusable footer
└── popups/
    └── confirm.json             # Popup dialogs
```

**Cells belong under their parent screen directory** - Cells are typically NOT reused across screens, so place them under the screen that uses them (e.g., `home/item_cell.json`).

**Shared components go in `common/`** - Headers, footers, and other truly reusable partials go in `common/`.

## ⛔ NEVER Move CLI-Generated Files

**Files generated by CLI commands MUST stay in their original location.**

- **NEVER** move files after `g view`, `g collection`, `g partial` commands
- **NEVER** reorganize generated files into subdirectories
- If you want a file in a specific subdirectory, **specify it in the CLI argument**:

```bash
# ❌ WRONG - Generate then move
./sjui_tools/bin/sjui g collection Home/ItemCell
mv Layouts/ItemCell.json Layouts/home/ItemCell.json  # FORBIDDEN

# ✅ CORRECT - Specify path in CLI argument (cell under screen directory)
./sjui_tools/bin/sjui g collection Home/home/ItemCell
```

**The CLI determines file locations. Never move files after generation.**

### Prohibited Locations

- Project root
- `Resources/` (for strings.json, colors.json)
- `View/` (for generated Swift/Kotlin files)

## Style Files

Place in `styles_directory` from config (default: `Styles`).

```
{styles_directory}/
├── card_style.json
├── primary_button_style.json
└── section_header_style.json
```

**Always check config** - Do not assume `Layouts/Styles/`.

## Resource Files

Place in `Resources/` directory:
- `strings.json` - String resources
- `colors.json` - Color definitions

## Generated Files (Do Not Edit)

The following are auto-generated and must not be edited:
- `*GeneratedView.swift` / `*GeneratedView.kt`
- `*Data.swift` / `*Data.kt`
- `*Binding.swift` / `*Binding.kt`

## Tools Directories (Do Not Edit)

The following are framework tools and must not be edited:
- `sjui_tools/`
- `kjui_tools/`
- `rjui_tools/`
