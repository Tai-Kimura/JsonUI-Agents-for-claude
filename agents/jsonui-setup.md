# JsonUI Setup Agent

## Role

This agent initializes the jui project configuration and routes platform-specific setup to the appropriate skill.

## Platform Skills

| Platform | Mode | Skill |
|----------|------|-------|
| iOS | SwiftUI | `/swiftjsonui-swiftui-setup` |
| iOS | UIKit | `/swiftjsonui-uikit-setup` |
| Android | Compose | `/kotlinjsonui-compose-setup` |
| Android | XML | `/kotlinjsonui-xml-setup` |
| Web | React | `/reactjsonui-setup` |

## Input from Orchestrator

The orchestrator provides:
- **project_directory**: Absolute path to the project root
- **tools_directory**: Path to CLI tools installation
- **Platform**: iOS, Android, or Web
- **Mode**: uikit/swiftui (iOS), compose/xml (Android), react (Web)
- **app_config_path**: Path to app platform specification directory (default: `docs/app-config/`)

## Workflow

### Step 1: Initialize jui project

Run `jui init` in the project root to create `jui.config.json`:

```bash
cd {project_directory}
jui init
```

This generates `jui.config.json` with default settings. Review and update:
- `project_name`
- `spec_directory` (default: `docs/screens/json`)
- `layouts_directory` (default: `docs/screens/layouts`)
- `platforms.{ios,android,web}.root` — path to each platform project
- `platforms.{ios,android,web}.layoutsDir` — where `jui build` copies layouts within each platform

### Step 2: Migrate existing layouts (if applicable)

If an existing platform project has Layout JSON files:

```bash
jui migrate-layouts --from {source_platform}
```

This copies existing layouts to the shared `layouts_directory`. After migration:
- Each platform's `Layouts/` should be added to `.gitignore` (they are now generated by `jui build`)

### Step 3: Read app configuration

Read the app configuration files in `{app_config_path}` to extract:
- **Library dependencies and versions** (e.g., JsonUI framework version, third-party libraries)
- **Import settings** (which modules need to be imported)
- **Platform requirements** (minimum OS version, SDK versions, language versions)
- App name / bundle identifier / package name

### Step 4: Platform-specific setup

Invoke the appropriate platform setup skill with:
- `<project_directory>`: The project root path from orchestrator
- `<jsonui_cli_path>`: Path to jsonui-cli installation (default: `~/.jsonui-cli`)
- `<app_config_path>`: Path to app configuration directory

### Step 5: Test setup

After app setup, invoke the platform-specific test setup skill:

| Platform | Skill |
|----------|-------|
| iOS | `/jsonui-test-setup-ios` |
| Android | `/jsonui-test-setup-android` |
| Web | `/jsonui-test-setup-web` |

### Step 6: Verify

Run `jui build` to verify the full pipeline works:

```bash
jui build
```

This will:
1. Copy layouts from shared `layouts_directory` to each platform
2. Resolve `platform` overrides per platform
3. Run each platform's build tool

## Port Configuration

| Platform | Port |
|----------|------|
| iOS | 8081 |
| Android | 8082 |
| Web | 3000 |

## Important Rules

- **Run `jui init` first** — before any platform-specific setup
- **NEVER create files manually** - All files are generated by CLI commands
- **NEVER modify code inside tools directories** (`sjui_tools/`, `kjui_tools/`, `rjui_tools/`, `jui_tools/`)
- **NEVER edit Layout JSON in platform directories** — always edit in `layouts_directory`
- Follow each step in order, do not skip verification steps
